Kimi K2.7-Code (Moonshot) — открытая модель для кода, только вендорские бенчмарки Kimi K2.7-Code (Moonshot) — open-weight coding model, vendor-benchmark only release

Moonshot выпустил (12 июня) открытую, специализированную на коде преемницу K2.6. Архитектура: MoE на 1T параметров, 32B активных, 384 эксперта (выбирается 8+1), контекст 256K, vision-энкодер MoonViT на 400M, обязательный неотключаемый режим «thinking», зафиксированные параметры сэмплинга. Веса доступны на Hugging Face (moonshotai/Kimi-K2.7-Code) под Modified MIT. Цена API $0,95 / $4,00 за 1M входных/выходных токенов.

Качество заявлений: все заголовочные цифры — собственные проприетарные бенчмарки Moonshot (Kimi Code Bench v2, Program Bench, MLS Bench Lite, MCP Atlas, MCPMark Verified). Флагманские заявления: +21,8% к K2.6 на Kimi Code Bench v2 и ~30% меньше reasoning-токенов. На 15 июня независимых сторонних цифр на стандартных публичных наборах (SWE-bench Verified/Pro, LiveCodeBench, Terminal-Bench, GPQA Diamond) нет, как и оценки Artificial Analysis Index (для сравнения: предшественница K2.6 — 54).

Вердикт триажа — порог не пересекает. По собственным цифрам Moonshot K2.7-Code уступает GPT-5.5 и Claude Opus 4.8 на большинстве код-бенчмарков (Kimi Code Bench v2 62,0 против 69,0 у GPT-5.5 / 67,4 у Opus 4.8; MLS Bench Lite 35,1 против 81,3 у Opus 4.8). Реальное преимущество — цена и токен-эффективность, а не способности. Флаг целостности eval (причина лога): разработчик Sugumaran Balasubramaniyan (платформа Hermes Agent) публично оспорил цифры — «при всём уважении, любая модель „улучшается“ на двузначные проценты на собственном тест-сете», — напомнив, что K2.6 набрала лишь 24% на независимом DeepSWE, лидируя на своих наборах, а K2.7-Code на DeepSWE не подавали. Хрестоматийная «заголовочная метрика без истории независимой проверки» — считать прирост направленным, не результатом.

Moonshot shipped (June 12) an open-weight, coding-specialized successor to K2.6. Architecture: 1T-param MoE, 32B active per token, 384 experts (8+1 selected), 256K context, a 400M MoonViT vision encoder, mandatory always-on "thinking" mode (non-disableable), locked sampling parameters. Weights are live on Hugging Face (moonshotai/Kimi-K2.7-Code) under a Modified MIT license. API pricing $0.95 / $4.00 per 1M input/output tokens.

Claim quality: all headline numbers are Moonshot's own proprietary benchmarks (Kimi Code Bench v2, Program Bench, MLS Bench Lite, MCP Atlas, MCPMark Verified). Flagship claims: +21.8% over K2.6 on Kimi Code Bench v2 and ~30% fewer reasoning tokens. As of June 15 there are NO independent third-party numbers on standard public suites (SWE-bench Verified/Pro, LiveCodeBench, Terminal-Bench, GPQA Diamond), and no Artificial Analysis Index score yet (for reference, predecessor K2.6 scored 54).

Triage verdict — does NOT clear the frontier threshold. By Moonshot's own numbers, K2.7-Code trails GPT-5.5 and Claude Opus 4.8 on most coding benchmarks (Kimi Code Bench v2 62.0 vs GPT-5.5 69.0 / Opus 4.8 67.4; MLS Bench Lite 35.1 vs Opus 4.8 81.3). The real selling point is price and token efficiency, not capability. Eval-integrity flag (why it's logged): developer Sugumaran Balasubramaniyan (Hermes Agent platform) publicly challenged the numbers — "respectfully, every model 'improves' double digits on its own test suite" — noting K2.6 scored only 24% on the independent DeepSWE benchmark while leading on Moonshot's own suites, and K2.7-Code has not been submitted to DeepSWE. A textbook "headline metric without an independent-eval story" — treat the gains as directional, not a result.
